# План интеграции МойСклад

1. Источник данных
   МойСклад становится главным источником для товаров, модификаций, цен, остатков, контрагентов и статусов заказов. Сайт хранит копию данных для витрины и личного кабинета.

2. Настройки аккаунта
   В МойСклад должны быть заведены организация, основной склад, услуга "Доставка", группы товаров, товары с модификациями, типы цен и статусы заказов. Для товара с фасовками используется схема: основной товар + модификации.

3. Поля связи
   В Payload добавляются поля `moyskladId` для типов, категорий, товаров, модификаций, клиентов и заказов. Это защищает от дублей и позволяет обновлять уже связанные сущности.

4. Отправка заказов
   После оформления заказа на сайте создается заказ покупателя в МойСклад. В заказ уходят контрагент, компания, позиции, модификации, количество, цена, скидка, доставка, комментарий и номер заказа сайта.

5. Контрагенты
   Перед отправкой заказа сайт ищет контрагента по `moyskladCounterpartyId` или email. Если контрагент не найден, он создается в МойСклад и привязывается к клиенту на сайте.

6. Товары и модификации
   Синхронизация товаров строится по `moyskladId`. Модификация МойСклад соответствует варианту фасовки на сайте. Если у товара нет модификаций, позиция может ссылаться на основной товар.

7. Цены и остатки
   Цены и доступность на сайте обновляются из МойСклад. Остатки подтягиваются по выбранному складу. Если остаток нулевой, вариант становится недоступным.

8. Статусы
   Статусы заказов подтягиваются из МойСклад и маппятся на статусы сайта: новый, подтвержден, счет выставлен, оплачен, в работе, готов, отгружен, доставлен, отменен.

9. Счета
   Если клиент хочет счета из МойСклад, сайт хранит номер/ссылку счета из МойСклад. Генерация PDF на сайте может остаться как резервный вариант или быть отключена позже.

10. Логи
    Все попытки синхронизации пишутся в таблицу `moysklad_sync_logs`: направление, сущность, локальный ID, ID МойСклад, статус, ошибка, дата.

11. Безопасный запуск
    Интеграция включается только через `MOYSKLAD_ENABLED=true`. Если МойСклад недоступен, заказ на сайте не ломается, а ошибка пишется в заказ и лог.

12. Тест
    Перед боевым запуском проверяется один товар с модификациями, один клиент, одна компания и один заказ. После сверки сумм, скидок и статуса включается синхронизация для всех заказов.
